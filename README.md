# 消えた鍵と最後の晩餐

スマートフォンブラウザ向け推理サスペンスノベルゲーム。

山間の高級ホテル「グランド・ハゲルヤ」で起きた事件を、探偵として調査し真相を解き明かす。

## ゲーム概要

- プロローグ + 全3章構成のテキストアドベンチャー
- スマートフォンブラウザでの操作に最適化（タッチ操作・セーフエリア対応）
- 選択肢による分岐、パズル（暗証番号入力・暗号解読・証拠整理）
- 手がかり収集、記憶の断片システム、ヒント表示、メモ機能
- 疑惑レベルシステム（プレイヤーの行動で変化）
- メタナラティブ演出（システムからの割り込みメッセージ）
- ターミナル風ブートシーケンス（ゲーム開始時の名前入力）
- **4種マルチエンディング**：
  - **BAD END** ── 誤った告発による捜査失敗
  - **NORMAL END** ── 状況証拠のみでの事件解決
  - **TRUE END** ── 決定的物証による完全解決
  - **SECRET END** ── 記憶の断片と高い疑惑レベルで到達する、真の結末
- エンディング後のプレイヤー心理タイプ診断（TRUE/SECRET END 時）
- CASE_02 予告演出（エンディング種別で演出が変化）
- クリア記録がタイトル画面に表示される（全4種の到達状況）

## 対応環境

### 推奨ブラウザ

スマートフォンブラウザでの動作を前提に設計されています。

- **iOS**: Safari 15 以降
- **Android**: Chrome 90 以降
- **PC**: Chrome / Safari / Firefox / Edge（最新版）

### スマホ最適化の対応内容

- `viewport-fit=cover` による全画面レイアウト
- `safe-area-inset` 対応（ノッチ・ダイナミックアイランド）
- `apple-mobile-web-app-capable` によるスタンドアロン表示対応
- タッチ操作とスクロールの区別（移動量・時間で判定）
- 横向き（ランドスケープ）時のコンパクトレイアウト
- アプリ切り替え後の音声自動再開（`visibilitychange` / `pageshow` / `focus`）

## 起動方法

### ローカルで遊ぶ

1. プロジェクトフォルダ内で HTTP サーバーを起動する

```bash
# Python 3
python3 -m http.server 8000

# Node.js (npx)
npx serve .
```

2. ブラウザで `http://localhost:8000` を開く

> `index.html` を直接ダブルクリックしても動作しますが、音声ファイルを配置している場合はサーバー経由でのアクセスを推奨します。

### スマホで遊ぶ

- 同一 Wi-Fi 内であれば `http://<PCのIPアドレス>:8000` でスマホからアクセス可能
- GitHub Pages で公開すればどこからでもアクセス可能（後述）

### 動作確認

サーバーを起動してブラウザで開いた後、以下を確認してください。

1. タイトル画面が表示され、ボタンが操作できる
2. 「はじめから」タップでブートシーケンス（ターミナル風の名前入力）が始まる
3. テキストがタイプライター風に表示され、タップで送れる
4. サウンドボタンで音声の ON/OFF を切り替えられる
5. ツールバーの各ボタン（保存・手がかり・ヒント・メモ）が開く

## 操作方法

| 操作 | 内容 |
|------|------|
| 画面タップ | テキスト送り / タイプ演出スキップ |
| 選択肢タップ | ストーリー分岐 |
| パズル入力 | テキスト入力 → 回答ボタン（Enter キーでも送信可） |
| 証拠整理 | 候補カードをタップで選択、スロットタップで取り消し |
| ツールバー | 保存 / 手がかり / ヒント / メモ |
| サウンドボタン | ミュート ON/OFF（設定は保持） |

## ゲームシステム詳細

### 手がかり収集

特定のシーンに到達すると手がかりを自動取得。ツールバーの「手がかり」から一覧を確認できる。

### 記憶の断片

ストーリー中に散りばめられた4つの記憶の断片。すべて集めることが SECRET END への条件の一つ。手がかり帳の「記憶の断片」セクションに表示される。

### 疑惑レベル

探索行動やストーリー進行で 0〜5 の範囲で変化。画面上部に「違和感」インジケーターとして表示される。SECRET END 到達には高い疑惑レベル（4以上）が必要。

### パズル

- **暗証番号入力**: 書斎で見つけたメモの数字を入力
- **暗号解読**: シーザー暗号の解読（アルファベットを一つ前にずらす）
- **証拠整理**: 候補カードから正しい証拠を正しい順序で3つ選択

各パズルにはインラインヒント機能があり、「ヒントを見る」ボタンで段階的にヒントが表示される。

### エンディング分岐条件

| エンディング | 条件 |
|------------|------|
| BAD END | 犯人指名で赤坂以外を選択 |
| NORMAL END | 赤坂を指名（薬品棚の物証なし、SECRET 条件未達） |
| TRUE END | 赤坂を指名 + 薬品棚の管理簿（物証）を発見済み |
| SECRET END | 赤坂を指名 + 記憶の断片4つすべて入手 + 疑惑レベル4以上 |

### プレイヤー心理タイプ診断

TRUE END または SECRET END 到達時、プレイ行動に基づいて4種の心理タイプに診断される。

- **THE MANIPULATOR** ── 操る者（素早い判断、少ないミス）
- **THE ARCHITECT** ── 設計者（徹底的な探索、メモ活用）
- **THE OBSERVER** ── 傍観者（じっくり考え、ヒント活用）
- **THE SAVIOR** ── 救済者（バランス型）

## audio フォルダについて

`assets/audio/` フォルダに音声ファイルを配置すると、合成音の代わりにファイル音声が再生されます。
ファイルが存在しない場合は Web Audio API による合成音で自動的にフォールバックするため、
**音声ファイルなしでもゲームは正常に動作します。**

### 対応ファイル一覧

| ファイル名 | 用途 |
|-----------|------|
| `assets/audio/bgm_title.mp3` | タイトル画面 BGM |
| `assets/audio/bgm_dark.mp3` | 本編 BGM（序盤） |
| `assets/audio/bgm_tension.mp3` | 本編 BGM（緊迫場面） |
| `assets/audio/bgm_reveal.mp3` | エンディング BGM |
| `assets/audio/se_tap.mp3` | テキスト送り SE |
| `assets/audio/se_choice.mp3` | 選択肢タップ SE |
| `assets/audio/se_clue.mp3` | 手がかり入手 SE |
| `assets/audio/se_correct.mp3` | パズル正解 SE |
| `assets/audio/se_wrong.mp3` | パズル不正解 SE |
| `assets/audio/se_chapter.mp3` | チャプターカード SE |

対応形式: mp3 / ogg / wav

## データ保存（localStorage）

ゲームの進行データはブラウザの `localStorage` に保存されます。

- **オートセーブ**: シーン移動ごとに自動保存
- **手動セーブ**: 3スロット
- **クリア記録**: 4種エンディングの到達記録
- **ミュート設定**: ON/OFF 状態

| キー | 内容 |
|------|------|
| `hageruya_auto` | オートセーブ |
| `hageruya_1` ~ `hageruya_3` | 手動セーブスロット |
| `hageruya_clear` | クリア記録（BAD / NORMAL / TRUE / SECRET） |
| `hageruya_mute` | ミュート設定 |

> プライベートブラウジングや localStorage が無効な環境でもエラーなく動作します（保存のみ無効になります）。

タイトル画面の「進行データをリセット」ボタンから、すべてのセーブデータとクリア記録を一括削除できます。

## GitHub Pages での公開方法

### 方法 A：ブラウザだけで公開する（初心者向け）

Git やターミナルを使わず、ブラウザ操作だけで公開できます。

#### 1. GitHub アカウントを作成する

- https://github.com にアクセス
- **Sign up** からアカウントを作成（すでにある場合はログイン）

#### 2. 新しいリポジトリを作成する

- 右上の **+** ボタン → **New repository** をクリック
- **Repository name** に `hageruya-case01` と入力
- **Public** を選択（GitHub Pages は Public リポジトリで無料）
- **Create repository** をクリック

#### 3. ファイルをアップロードする

- 作成されたリポジトリのページで **uploading an existing file** リンクをクリック
- 以下のファイル・フォルダをすべてドラッグ＆ドロップする：
  - `index.html`
  - `css/` フォルダ
  - `js/` フォルダ
  - `assets/` フォルダ（音声ファイルがある場合）
- 下部の **Commit changes** をクリック

#### 4. GitHub Pages を有効にする

- リポジトリページの上部メニューから **Settings** をクリック
- 左サイドバーの **Pages** をクリック
- **Source** で **Deploy from a branch** を選択
- **Branch** で **main** を選択、フォルダは **/ (root)** のまま
- **Save** をクリック

#### 5. 公開 URL を確認する

- 1〜2分待ってから **Settings → Pages** を再読み込み
- 上部に公開 URL が表示される：
  ```
  https://<ユーザー名>.github.io/hageruya-case01/
  ```
- この URL をスマホのブラウザで開けばすぐにプレイ可能

---

### 方法 B：ターミナル（Git）で公開する

```bash
# 1. プロジェクトフォルダに移動
cd hageruya-case01

# 2. Git リポジトリを初期化
git init
git add .
git commit -m "Initial commit"

# 3. GitHub にリモートリポジトリを作成済みの状態で接続
git remote add origin https://github.com/<ユーザー名>/hageruya-case01.git
git branch -M main
git push -u origin main
```

その後は **方法 A の手順 4〜5** と同じく Settings → Pages で有効化する。

---

### 公開後の更新方法

ファイルを修正した場合：

- **方法 A**: GitHub のリポジトリページでファイルを選択 → 鉛筆アイコンで編集 → **Commit changes**
- **方法 B**: ローカルで修正後 `git add . && git commit -m "修正内容" && git push`

更新は数分以内に公開 URL に反映されます。
